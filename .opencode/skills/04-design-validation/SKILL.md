# 04-design-validation Skill

**Objective:**
Validate the logical database schema to ensure it correctly answers the original business requirements.

**Instructions for the Agent:**
1. Read `req/business-requirement.md` (original requirements) and `outputs/03-logical-design-G11.md` (schema).
2. Evaluate if the relations and constraints correctly represent the ERD and business rules.
3. Check for any missing entities, attributes, or constraints.
4. Identify if appropriate primary keys and relationships have been used.
5. Provide a summary of the validation: list passing checks and note any elements that required adjustment or specific assumptions.

**Output:**
Create or update `outputs/04-design-validation-G11.md`.
